## 2024-11-20 - Windows Forms Accessibility Standards
**Learning:** WinForms does not use web-based ARIA attributes (like `aria-label`). Instead, screen readers and accessibility tools rely on the `AccessibleName` property, while sighted users benefit from the `ToolTip` component for icon-only interactive elements.
**Action:** Always set both `AccessibleName` and add a `ToolTip` (using `_toolTip.SetToolTip()`) for any icon-only `Button` or control in Windows Forms applications to ensure both full accessibility and a good UX.
## 2024-12-05 - Windows Forms Icon Buttons Accessibility
**Learning:** Icon-only close buttons ("✕") in Windows Forms applications lacked tooltips and accessible names. Unlike web applications where `aria-label` is used, Windows Forms requires setting the `AccessibleName` property for screen readers and explicitly adding tooltips (e.g. using a `ToolTip` component or an extension method like `ToolTipText`) to provide visual context for sighted users.
**Action:** When adding or updating icon-only buttons in Windows Forms, consistently provide both `AccessibleName` and visual tooltips to ensure a fully accessible and intuitive UX. The `ButtonExtensions.ToolTipText` helper can be made `public` and used for clean syntax.
